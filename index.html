正则表达式之使用元字符

发表于 2017-10-30 | 分类于 正则表达式

正则表达式必知必会之使用元字符

标签（空格分隔）： 正则表达式

对特殊字符进行转义

因为元字符在正则表达式里有着特殊的含义，所以这些字符无法代表它们本身。在元字符前面加一个反斜杠就可以对它进行转义：转义序列.将匹配.本身，转义序列[将匹配[本身。

文本：

正则表达式：

cat test.txt | egrep 'myArray\[0\]'

结果：

if (myArray[0] == 0) {

注意:
任何一个元字符都可以通过加一个反斜杠()作为前缀进行转义。配对的元字符（比如[]）不用作元字符时必须被转义，否则正则表达式分析器很可能会抛出一个异常。

对元字符进行转义需要用到\字符，这意味着\也是一个元字符，对其进行转义时需要使用\。

文本：

\home\ben\sales

正则表达式：

cat test.txt | grep '\\'

结果：

\ \ \

匹配空白字符

元字符大致分为两种：一种是用来匹配文本的（比如 . ），另一种是正则表达式的语法所要求的（比如[和]）。

元字符 说明
[\b] 回退（并删除）一个字符（Backspace键）
\f 换页符
\n 换行符
\r 回车符
\t 制表符（Tab键）
\v 垂直制表符

匹配特定的字符类别

匹配数字（与非字符）

元字符 说明
\d 任何一个数字字符（等价于[0-9]）
\D 任何一个非数字字符（等价于[^0-9])

文本：

正则表达式：

cat test.txt | grep -P 'myArray\[\d\]'

结果：

myArray[0]

注意：
正则表达式的语法是区分字母大小写的。\d匹配数字，\D与\d的含义刚好相反。

匹配字母和数字（与非字母和数字）

字母和数字，A到Z（不分大小写）、数字0到9，再加上下划线(_)，是另一种常用的字符集合。这些字符常见于各种名字中，如（文件名、目录名、变量名、数据库对象）。

元字符 说明
\w 任何一个字母数字字符（大小写均可）或下划线字符（等价于[a-zA-Z0-9_]）
\W 任何一个非字母数字或非下划线(等价于)[^a-zA-Z0-9_]

文本：

正则表达式：

cat test.txt | grep -P '\w\d\w\d\w\d'

结果：

A1C2E3

M1B4F2

H1H2H2

匹配空白字符

元字符 说明
\s 任何一个空白字符（等价于[\f\n\r\t\v])
\S 任何一个非空白字符（等价于[^\f\n\r\t\v]）

注意：
用来匹配退格字符的[\b]元字符是一个特例：它不在类元字符\s的覆盖范围内，当然也就没有排除在类元字符\S的覆盖范围外。

匹配十六进制或八进制数值

  1. 使用十六进制值
    在正则表达式里，十六进制（逢16进1）数值要用前缀\x来给出。比如说，\x0A对应于ASCII字符10（换行符），其效果等价于\n。
  2. 使用八进制值
    在正则表达式里，八进制（逢8进1）数值要用前缀\0来给出，数值本身可以是两位或三位数字。比如说，\011对应于ASCII字符9（制表符），其效果等价于\t。

使用POSIX字符类

POSIX字符类是许多（不是所有）正则表达式实现都支持的一种简写形式。

字符类 说明
[:alnum:] 任何一个字母或数字（等价于[a-zA-Z0-9])
[:alpha:] 任何一个字母（等价于[a-zA-z]）
[:blank:] 空格或制表符[\t ]
[:cntrl:] ASCII控制字符（ASCII0到31），再加上ASCII127
[:digit:] 任何一个数字（等价于[0-9]）
[:graph:] 和[:print:]一样，但不包括空格
[:lower:] 任何一个小写字母（等价于[a-z]）
[:print:] 任何一个打印字符
[:punct:] 既不属于[:alnum:]也不属于[:cntrl:]的任何一个字符
[:space:] 任何一个空白字符，包括空格（等价于[^\f\n\r\t\v]）
[:upper:] 任何一个大写字母(等价于[A-Z])
[:xdigit:] 任何一个十六进制数字（等价于[a-fA-F0-9]）

文本：

正则表达式：

cat test.txt | grep -P '#[[:xdigit:]][[:xdigit:]][[:xdigit:]][[:xdigit:]][[:xdigit:]][[:xdigit:]]'

结果：

#336633 #FFFFFF

注意：这里使用的模式以[[开头、以]]结束（两对方括号）。这是使用POSIX字符类所必须的。POSIX字符类必须括在[:和:]之间，我们使用的POSIX字符类是:xdigit:。外层的[和]字符用来定义一个字符集合，内层的[和]字符是POSIX字符类本身的组成部分。

正则表达式之匹配一组字符

发表于 2017-10-24 | 分类于 正则表达式

匹配一组字符

标签（空格分隔）：正则表达式

匹配多个字符中的某一个

在正则表达式里，可以使用元字符[和]来定义一个集合。在使用[和]定义的字符集合里，这两个元字符之间的所有字符都是该集合的组成部分，字符集合的匹配结果是能够与该集合里的任意一个成员相匹配的文本。

文本：

sales1.xls

orders3.xls

sales2.xls

sales3.xls

apac1.xls

europe2.xls

na1.xls

na2.xls

sa1.xls

sa2.xls

正则表达式：

cat test.txt | grep '[ns]a.\.xls'

结果：

na1.xls

na2.xls

sa1.xls

sa2.xls

利用字符集合区间

文本：

sales1.xls

orders3.xls

sales2.xls

sales3.xls

apac1.xls

europe2.xls

sam.xls

na1.xls

na2.xls

sa1.xls

sa2.xls

正则表达式：
cat test.txt | grep '[ns]a[0-9]\.xls'

结果：

na1.xls

na2.xls

sa1.xls

sa2.xls

在使用正则表达式时，会频繁地使用一些字符区间（0~9、A~Z等）。为了简化字符区间定义，正则表达式提供了一个特殊的元字符 - 来使用。
字符区间不仅限于数字，以下这些也是合法的字符区间。

  • A-Z，匹配从A到Z的所有大写字母。
  • a-z，匹配从a到z的所有小写字母。
  • A-F，匹配从A到F的所有大写字母。
  • A-z，匹配从ASCII字符A到z的所有字母。该模式一般不常用，因为还包含[和^等在ASCII字符表里排列在Z和a之间的字符。

注意:

-（连字符）是一个特殊的元字符，作为元字符只能用在[和]之间。在字符集合以外的地方，-只是一个普通字符，只能与-本身相匹配。因此在正则表达式里不需要被转义。

在同一个字符集合里可以匹配多个字符区间。比如说下面的模板可以匹配任何一个字母（无论大小写）或数字。
[A-Za-z0-9]

取非匹配

取非匹配，是指除了字符集合里的字符外，其他字符都可以匹配。使用元字符^来表明你想对一个字符集合进行取非匹配。

文本：

sales1.xls

orders3.xls

sales2.xls

sales3.xls

apac1.xls

europe2.xls

sam.xls

na1.xls

na2.xls

sa1.xls

sa2.xls

正则表达式：
cat test.txt | grep '[ns]a[^0-9]\.xls'

结果：
sam.xls

小结

元字符[和]用来定义一个字符集合。其含义是必须匹配集合中的字符之一。定义一个字符集合的方法有两种：一是把所有的字符全都列举出来；二是利用元字符-以字符区间的方式给出。字符集合可以用元字符^来求非。这将把给定的字符集合强行排除在匹配操作以外。除了该字符集合里的字符，其他字符都可以被匹配。

正则表达式之匹配单个字符

正则表达式入门

标签（空格分隔）： 正则表达式

正则表达式用途

正则表达式（regular expression）是一种工具。主要有以下应用场景：

  • 搜索文件中的单词Car（不区分大小写）
  • 替换源代码中的单词size为isize
  • 检查电子邮箱地址是否合法
  • 列出包含Application字样的文件
  • 搜索特定的文本
  • 修改数据以制表符为分隔符为CSV格式

如何使用正则表达式

用正则表达式进行搜索

文件searchtest.txt

正则表达式

cat test.txt | grep '\b[Cc][Aa][Rr]\b'

结果

car

Car

CAR

正则表达式进行替换

正则表达式是什么

正则表达式是用来匹配和处理文本的字符串，正则表达式是用正则表达式语言创建的。正则表达式内置于其他语言中，几乎所有的语言或工具都支持正则表达式。

匹配单个字符

文本

正则表达式

cat text.txt | grep 'my'

匹配结果

Hello, my name is Ben.Please visit

my website at http://www.forta.com

匹配纯文本

正则表达式

cat text.txt | grep 'Ben'

匹配结果

Hello, my name is Ben.Please visit

my website at http://www.forta.com

多个匹配结果

字母大小写

正则表达式是区分字母大小写的，所以Ben不匹配ben。

匹配任意字符

在正则表达式里，特殊字符（或字符集合）用来给出要搜索的东西。字符 . （英文句号）可以匹配任何一个单个字符。. 字符可以用来匹配任何单个的字符、字母、数字甚至是 . 字符本身。

文本

正则表达式

cat test.txt | grep 'sales.'

结果

sales.xls

sales1.xls

orders3.xls

sales2.xls

sales3.xls

apac1.xls

na1.xls

na2.xls

注意：正则表达式又称为模式。它可以用来匹配包含着字符串内容的模式。匹配的并不总是整个字符串，而是与某个模式相匹配的字符，即使它们是整个字符串的一部分。

在同一个正则表达式里允许使用多个 . 字符，它们既可以连续出现，也可以间隔着出现在模式的不同位置。

匹配特殊字符

. 字符在正则表达式里有着特殊的含义。如果在模式里需要一个 . 字符，就需要在前面加一个 \ 来进行转义。
